=================================================

SOURCE: https://learn.microsoft.com/en-us/microsoft-copilot-studio/agent-extend-action-mcp
SOURCE: https://devblogs.microsoft.com/microsoft365dev/build-declarative-agents-for-microsoft-365-copilot-with-mcp/

KEY FINDINGS:
1. Copilot Studio supports MCP natively (since Ignite 2025)
2. You can build "declarative agents" for M365 Copilot using MCP
3. MCP allows connecting to existing knowledge servers and APIs
4. GitHub Copilot also supports MCP for extending capabilities
5. VS Code Agent Skills use MCP for specialized capabilities
6. Custom engine agents allow building AI assistants with custom models

THIS IS THE INTEGRATION PATH FOR UWS → COPILOT:
- Build uws as an MCP server
- Copilot Studio connects to uws via MCP
- All 12,000-20,000 unified operations become Copilot skills
- The Alexandria spec becomes a declarative agent definition
- The constitutional principles become agent governance rules
