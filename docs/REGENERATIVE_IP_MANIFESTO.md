## The Problem: Value Extracted Into Thin Air

Every line of code ever shipped carries two invisible price tags.

The first is the one you see: the salary, the contract rate, the enterprise licence fee.
The second is the one nobody tracks: the *hard-won judgment* that decided which 200 lines
of AI output were worth keeping and which 800 were garbage.  That judgment — the Human-In-
The-Loop (HITL) review — is precisely where economic value is created.  It is also precisely
where traditional C-Suite economics extract that value and attribute it to the company, the
model vendor, or the CI/CD pipeline, rather than to the human whose expertise made it real.

This manifesto describes how `uws` solves that problem permanently and cryptographically.

---

## The Solution: GitHub as the Root IP Ledger

Git commits are the most durable, most widely-audited, most legally-defensible artefact in
software history.  Every contribution is timestamped, signed, and immutable once pushed.
GitHub is already the *de facto* global registry for code IP.

What has been missing is a lightweight semantic layer that answers:

> *Who is the primary economic driver of this commit, and in what proportion?*

The `uws ip` subsystem provides exactly that layer — without requiring a new blockchain,
a new token, or a new platform.  It works on top of the infrastructure that already exists.

---

## How It Works: The ProvenanceTrailer

When a developer calls `uws ip sign`, the CLI:

1. **Resolves the current HEAD commit SHA** via `git rev-parse HEAD`.
2. **Builds a `ProvenanceTrailer`** — a structured JSON record containing:
   - `human_weight` — the fraction of value attributed to human HITL work (e.g. `0.90`).
   - `ai_weight` — the fraction attributed to AI generation (e.g. `0.10`).
   - `revenue_split` — who gets paid and at what percentage, and through which ledger.
   - `hitl_signature` — a cryptographic signature proving the human reviewed and approved
     the asset before it was committed.
3. **Writes the trailer as a git note** (`git notes add`) so it is permanently attached to
   the commit in the repository's object store without rewriting history.

The result is a tamper-evident, auditable record that can be read by any downstream tool —
a GitHub Action, a smart contract, a legal compliance system, or the `uws ip monetize` stub
— to determine how revenue generated by that IP should be distributed.

## The First-Mover Advantage

GitHub's unique position as the world's largest code host gives it a structural advantage
that no Layer-2 blockchain or third-party IP registry can replicate:

| Property | Traditional IP Registry | GitHub + `uws ip` |
|---|---|---|
| Timestamp authority | Notary / timestamp server | Git commit graph |
| Tamper evidence | Certificate chain | SHA-1 / SHA-256 commit hash |
| Human verification | Wet signature | HITL cryptographic signature |
| Distribution | Postal / manual | Git push + `git notes push` |
| Payout rails | Bank wire / escrow | GitHub Sponsors / Krakoa ledger |
| Cost to the creator | Hundreds of dollars | One `uws ip sign` command |

The cost to GitHub to support this is a short HITL review step before merge.
The benefit to creators is that their economic contribution is permanently on the record.

## Regenerative vs. Extractive Economics

Traditional IP economics are *extractive*: value flows upward through corporate hierarchies
and is attributed to whoever holds the equity, not whoever generated the insight.

Regenerative IP economics are *additive*: every review, every correction, every architectural
decision that makes AI-generated code actually useful is tracked and rewarded.  Value is not
extracted into thin air; it is redirected back to the people who created it.

The `revenue_split` field in the `ProvenanceTrailer` encodes this directly.  A typical
record might read:

```json
{
  "human_weight": 0.90,
  "ai_weight": 0.10,
  "revenue_split": {
    "creator_id": "alice@example.com",
    "creator_pct": 80,
    "ledger": "github-sponsors"
  }
}
```

This says: *Alice did 90% of the work.  80% of any downstream revenue goes to Alice via
GitHub Sponsors.  The remaining 20% goes to whichever secondary parties are registered.*

No board of directors.  No equity dilution.  No C-Suite 101 required.

## Architecture Diagram

```
Developer workstation
  │
  ├─ writes code + reviews AI output (HITL)
  │
  └─ uws ip sign --hitl-weight=0.9 --ai-weight=0.1
         │
         ├─ ProvenanceTrailer (JSON)
         │     ├── commit_sha: abc1234…
         │     ├── human_weight: 0.9
         │     ├── ai_weight: 0.1
         │     ├── revenue_split: { creator: alice, pct: 80, ledger: github-sponsors }
         │     └── hitl_signature: { reviewer: alice, sig: <hex>, algo: FNV1a-256 }
         │
         └─ git notes add -m <ProvenanceTrailer JSON>
               │
               └─ stored permanently in the git object store
                     │
                     ├─ uws ip monetize  → payout ledger registration
                     ├─ GitHub Sponsors  → live revenue distribution
                     └─ Legal compliance → audit trail
```

## Quick Start

```bash
# Sign the current HEAD commit with 90% human / 10% AI attribution
uws ip sign --hitl-weight=0.9 --ai-weight=0.1 --reviewer=you@example.com

# Preview what the sign command would produce (no git note written)
uws ip sign --dry-run

# Register the signed IP with the default payout ledger
uws ip monetize

# Register with a specific ledger
uws ip monetize --ledger=github-sponsors
```
